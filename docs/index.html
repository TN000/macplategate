What it does

A focused stack — entry / exit camera, plate recognizer, gate trigger. Nothing else.

Native RTSP + HEVC

Custom 200-line H.265 RTSP client + RFC 7798 depacketizer. VideoToolbox decodes on the media engine into IOSurface-backed buffers. No FFmpeg, no go2rtc, no relay daemon.

ANE-accelerated OCR

Apple Vision .accurate runs on the Neural Engine — free with every Apple Silicon Mac. Optional secondary FastPlateOCR ONNX engine for cross-engine consensus.

5-layer dedup

Exact / Levenshtein-1 / Levenshtein-2 / ambiguous-glyph matrix / fragment-prefix. Built from real production misread incidents, covered by 301 unit tests.

Shelly Pro 1 webhook

Fires /rpc/Switch.Set with Digest auth on whitelist match. Two independent Shelly devices supported (entry / exit), each with its own pulse duration.

Local web UI

HTTPS + Basic Auth + IP whitelist + brute-force throttling. Manual gate control, recents view, statistics — without leaving your laptop. Off by default.

Local storage

SQLite + JSONL audit log + JPEG snapshots, all on the Mac. GDPR-trivial: data never leaves the device. 0600 file perms, 30-day retention default.

Why a native macOS stack

The platform leverage is the entire point. Every component below ships with macOS — there is no third-party runtime, no GPU driver, no FFmpeg fork to maintain.

Apple Neural Engine

Vision .accurate runs on the ANE for free. Sub-50 ms per frame on M4. No CUDA, no model conversion, no per-image cost.

VideoToolbox + IOSurface

HEVC decode lands in a CVPixelBuffer the ANE and Metal can read with zero copies. End-to-end GPU path from media engine to OCR.

Network.framework

Native RTSP client in 330 lines of Swift. No FFmpeg avformat, no librtsp, no go2rtc relay. Direct NWConnection over TCP with Digest auth.

Power profile

Mac mini idles at <5 W, peaks under 30 W with two cameras at 10 fps. Linux + Intel-NUC equivalents draw 3–5× more for the same throughput.
